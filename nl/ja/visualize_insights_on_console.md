# コンソールでの洞察の視覚化
{: #visualize_insights_on_console}

MobileFirst Analytics Console で、分析レポートの表示と構成、アラートの管理、およびクライアント・ログの表示を行うことができます。 左側のナビゲーションで**「Analytics Console」**をクリックすることによって、Mobile Foundation Operations Console から Analytics Console を起動します。

Analytics Console が起動され、デフォルトのダッシュボードが表示されます。 既にログおよび分析データをサーバーに送信しているクライアント・アプリケーションがある場合、関連レポートが作成され、表示されます。 収集された分析データをダッシュボードで確認できます。 この分析データは、アプリケーションの異常終了、アプリケーション・セッション、およびサーバー処理時間に関連している可能性があります。 さらに、カスタム・グラフの作成やアラートの管理も行うことができます。

モバイル分析の概要ビューに加えて、クライアント・ログやキャプチャーされたクライアント異常終了データに対して未加工検索を行う機能も分析フィーチャーに組み込まれています。さらに、Mobile Analytics に送り込まれるクライアント API 関数呼び出しで明示的に提供される追加データに対する未加工検索機能も組み込まれています。

## アプリケーション・データのモニター
{: #monitoring_app_data}

Mobile Analytics には、モバイル・アプリケーションのモニタリングや分析を行う機能が備えられています。 Mobile Analytics クライアント SDK を使用して、アプリケーション・ログを記録したりデータをモニターしたりできます。 開発者は、このデータを Mobile Analytics サービスに送信するタイミングを制御できます。 データが Mobile Analytics に配信されると、Mobile Analytics コンソールを使用して、モバイル・アプリケーション、デバイス、およびアプリケーション・ログに関する分析の洞察を得ることができます。

引き出すことのできる洞察には以下のようなものがあります。

**定義済みメトリック**

定義済みメトリックを使用して、以下のような疑問に答えることができます。
* 新規ユーザーは何人いるか?
* アプリケーションをアクティブに使用している人は何人いるのか?
* ユーザーはどのくらいの頻度でアプリケーションを使用しているのか?
* ユーザーはどんな時刻にアプリケーションを使用しているのか?
* ユーザーに好まれているデバイス・モデルは何か?
* レガシー・オペレーティング・システムのサポートをいつ廃止すべきか?
* パフォーマンスの問題が発生しているアプリケーションはどれか?

**カスタム・イベント**

独自のカスタム・イベントを追加することにより、以下のような疑問に答えることができます。
* 最も使用されているフィーチャーと最も使用されていないフィーチャーは何か?
* ユーザーはアプリのどこから入ってどこで終わらせているか?
* ユーザーが最もよく表示するアクティビティーは何か?
* ユーザーはアプリのワークフロー (例えば変換ファネル) を終了しているか?

## 視覚化できるレポート: ユーザー
{: #reports_visualized_users}

このレポートでは、指定された日付範囲内にアプリケーションを使用したアクティブ・ユーザーの数を示すグラフが表示されます。 また、レポートには、新規ユーザー (指定された日付範囲内に初めてアプリケーションを使用する固有ユーザー) の数も示されます。
グラフは、アプリケーション名、オペレーティング・システム、またはオペレーティング・システムのバージョンでフィルタリングできます。

## 視覚化できるレポート: セッション
{: #reports_visualized_sessions}

このレポートでは、指定された日付範囲内のアプリケーション・セッションを示すグラフが表示されます。 アプリケーションをデバイスのフォアグラウンドに移動すると、セッションが記録されます。 グラフは、アプリケーション名、オペレーティング・システム、またはオペレーティング・システムのバージョンでフィルタリングできます。

## 視覚化できるレポート: ネットワーク要求
{: #reports_visualized_network_requests}

Mobile Analytics コンソールでアプリケーションのネットワーク要求データを視覚化します。

以下の測定値のデータを確認できます。

**往復時間** - アプリがネットワーク要求を行うのにかかる時間の長さをミリ秒単位で定義します。
**要求カウント** - 1 つのアプリによるネットワーク要求の頻度を表示します。 データは平均値としても表示されます。
グラフは、アプリケーション名、オペレーティング・システム、またはオペレーティング・システムのバージョンでフィルタリングできます。

## 視覚化できるレポート: 異常終了
{: #reports_visualized_crashes}

Mobile Analytics コンソールでアプリケーションの異常終了に関する情報を参照すると、アプリケーションのモニターやトラブルシューティングに役立ちます。

「異常終了」ページの**「異常終了の概要」**表に、次のデータ列が表示されます。

**アプリケーション**: アプリケーション名<br/>
**異常終了 (Crashes)**: 当該アプリケーションの異常終了の総数<br/>
**合計使用回数 (Total Uses)**: ユーザーが当該アプリケーションを開いて閉じた合計回数<br/>
**異常終了率 (Crash Rate)**: 1 回の使用当たりの異常終了のパーセンテージ<br/>
「異常終了」表で、アプリケーションの異常終了に関する情報を即座に確認できます。 「異常終了 (Crashes)」棒グラフは、異常終了の経時的なヒストグラムを表示します。<br/>

クラッシュ・データの表示方法は 2 とおりあります。

1.  異常終了率の表示 (Display crash rate): 経時的な異常終了率
2.  合計異常終了数の表示 (Display total crashes): 経時的な合計異常終了数

## 視覚化できるレポート: トラブルシューティング
{: #reports_visualized_troubleshooting}

Mobile Analytics コンソールの**「トラブルシューティング」**ページでは、**「異常終了の要約 (Crash Summary)」**表を使用して、アプリ異常終了の詳細が表示されます。

**「異常終了サマリー (Crash Summary)」**表はソート可能です。この表には、以下のデータ列が含まれます。

* 異常終了
* デバイス
* 最後の異常終了
* アプリケーション
* OS
* メッセージ

項目の横の + アイコンをクリックして、**「異常終了の詳細」**表を表示します。この表には、次の列が含まれています。

* 異常終了した時刻
* アプリケーション・バージョン
* OS バージョン
* デバイス・モデル
* デバイス ID
* ダウンロード: 異常終了につながったログをダウンロードするためのリンク

**「異常終了の詳細」**表内の任意の項目を展開して、スタック・トレースなどの詳細を表示します。

**「異常終了の要約 (Crash Summary)」**表のデータは、重大レベルのアプリ・ログの照会によって取り込まれます。 アプリケーションで重大なアプリケーション・ログを収集していない場合、データはありません。
{: note}


## 視覚化できるレポート: ユーザー・フィードバック
{: #reports_visualized_userfeedback}

ユーザー・フィードバックは、Mobile Analytics を使用したアプリ内フィードバック分析を提供します。
Mobile Analytics のこの機能を使用して、以下を行なうことができます。
* **ユーザーおよびテスター**は、アプリケーションを実行および使用するときに、アプリケーションからのフィードバックおよびバグ・レポートを記録して送信できます。
* **アプリ所有者**は、このコンテキストの豊富なユーザー・フィードバックにより、アプリケーションのユーザー・エクスペリエンスをより深く理解できます。
* ただし、**開発者** は、バグやフィーチャー・ギャップを診断して修正するための正確なアプリケーション・コンテキストを受け取ります。

### ユーザー・フィードバックの有効化
{: #enable_user_feedback}

以下のステップを実行して、モバイル・アプリケーションがユーザー・フィードバックを取り込めるようにします。

#### アプリのインスツルメント
{: #instrument_app}

* フィードバック・モードを開始するようにモバイル・アプリを装備します。 API `Analytics.triggerFeedbackMode();` を呼び出して、フィードバック・モードを起動します。 <!--For more information, refer to the documentation [here](instrument_an_app.html)-->.
* この API は、ボタン、メニュー・アクション、ジェスチャーなどのアプリケーション・イベントで呼び出すことができます。

#### ユーザー・フィードバックの受信
{: #receive_feedback}

* アプリケーションのユーザーおよびテスターは、フィードバック用に装備されたアプリケーション・アクションをトリガーすることによって、フィードバック・モードに切り替えることができます。
* フィードバック・モードでは、画面キャプチャーと共に豊富なコンテキスト・フィードバックを収集して Mobile Analytics に送信することができます。

#### ユーザー・フィードバックの分析
{: #analyze_feedback}

* Mobile Analytics は、モバイル・アプリケーションから送信されるリッチ・コンテキスト・フィードバックを受信して統合します。
* Mobile Analytics コンソールにログオンし、**「ユーザー・フィードバック」**オプションを選択してフィードバックを表示します。
* アプリ所有者は、フィードバックを確認し、コメントを追加したり、フィードバックにレビュー状況のタグを付けたりすることができます。 通常、コメントは、フィードバックに対処するために作成される Git 問題へのリンクなどの予定アクションであったり、フィードバックに対するアクションが不要である理由を記述する文であったりします。
* レビュー状況を使用すると、さまざまなレビュー状況のオプションのいずれかでフィードバックをカテゴリー化することにより、フィードバックを効率的に管理できます。
